3 Methods for solving recurrences

  • solving recurrences = obtaining asymptotic ΘΘ or OO bounds on the solution

1. The Substitution method

  • we guess a bound and then use mathematical induction to prove our guess correct

2. The recursion-tree method

  • converts the recursion into a tree whose nodes represent the costs incurred at various levels of the recursion
  • we use techniques for bounding summations to solve the recurrence

3. The master theorem

  • provides bounds for recurrences of the form:
    T(n)=a⋅T(n/b)+f(n)T(n)=a⋅T(n/b)+f(n)

    , where a≥1a≥1, b>1b>1, and f(n)f(n) is a given function, arise frequently.

Equations of this kind characterize divide-and-conquer algotithms that create a subproblems, each of which is 1/b the size of the original problem, and in which the divide and combine steps together take f(n) time

Technicalities in recurrences

For recurrences like

  • we omit the boundary conditions (T(n)=Θ(1)T(n)=Θ(1), for sufficiently small n)
  • →→ we assume that T(n)T(n) is constant for small n

We simply write:

T(n)=2⋅T(n/2)+Θ(n)T(n)=2⋅T(n/2)+Θ(n)

The substitution method

1. Guess the form of the solution

2. Use mathematicalinduction to find the constants and show that the solution works

The recursion-tree method (RTM)

  • each node reprsents the cost of a single suboroblem somewhere in the set of recursive function invocations
  • we sum the costs within each level of the tree to obtain a set of per-level costs
  • we sum all the per-level costs to determine the total costs of all levels of the recursion

A recursion tree is best used to generate a good guess, which you can then verify by the substitution method

Intermission: Tree basics

Example Tree

Height of a node

= the number of edges on the longest path between the node and a leaf.

The height of a tree = the height of its root

Depth of a node

= the number of edges from the node to the root

  • the depth of the root is always 0

Levels of a tree

= 1 + the number of edges between a node and the root (i.e. Depth + 1)

The Master Theorem

  • provides a ‘recipe’ to solve recurrences of the form:
    T(n)=a⋅T(n/b)+f(n)T(n)=a⋅T(n/b)+f(n)

    , where a≥1a≥1, b>1b>1, and f(n)f(n) is an asymptotically positive function.
  • the recurrence of this form (see above), describes the running time of an algorithm that divides the original problem of size nn into aa subproblems of size n/bn/b each, where aa and bb are positive constants
    • to be more accurate: we interpret n/bn/b to mean either ⌊n/b⌋⌊n/b⌋ or ⌈n/b⌉⌈n/b⌉
  • →→ thus the aa subproblems are solved recursively, ech in time T(n/b)T(n/b)
  • the function f(n)f(n) describes the cost of the Divide and Conquer steps combined

In the case that the above described characteristics are given, then T(n)T(n) has the following asymptotic bounds:


1. If f(n)=O(nlogba−ϵ)f(n)=O(nlogb⁡a−ϵ), for some constant ϵ>0ϵ>0 →→ then T(n)=Θ(nlogba)T(n)=Θ(nlogb⁡a)


2. If f(n)=Θ(nlogba)f(n)=Θ(nlogb⁡a) →→ then T(n)=Θ(nlogba⋅log2n)T(n)=Θ(nlogb⁡a⋅log2⁡n)


3. If f(n)=Ω(nlogba+ϵ)f(n)=Ω(nlogb⁡a+ϵ), for some constant ϵ>0ϵ>0, and if a⋅f(n/b)≤c⋅f(n)a⋅f(n/b)≤c⋅f(n) for some constant c<1c<1 and all suffieciently large nn, then T(n)=Θ(f(n))T(n)=Θ(f(n))

  • in each of the 3 cases above we compare the function f(n)f(n) with the function nlogbanlogb⁡a
  • the larger of the two functions determines the solution to the recurrence
  • in Case 1 the function nlogbanlogb⁡a is larger
    • to be more accurate: f(n)f(n) mus be polynomially smaller than nlogbanlogb⁡a:
    • f(n)f(n) must be asymptotically smaller than nlogbanlogb⁡a, by a factor of nϵnϵ for some constant ϵ>0ϵ>0
  • in Case 3 the function f(n)f(n) is larger
    • to be more accurate: f(n)f(n) mus be polynomially larger than nlogbanlogb⁡a:
    • f(n)f(n) must be asymptotically larger than nlogbanlogb⁡a, by a factor of nϵnϵ for some constant ϵ>0ϵ>0
  • in Case2 the two functions are the same size and we multiply by a logarithmic factor

The 3 cases do not cover all the possibilities for f(n)f(n), e.g.:

  • when f(n)f(n) is smaller than nlogbanlogb⁡a, but not polynomially smaller
  • when f(n)f(n) is larger than nlogbanlogb⁡a, but not polynomially larger
  • when the regularity condition of case 3 fails to hold

→→ In these cases we cannot make use of the master theorem!

Using the master theorem:

Example 1:
Given recurrence:

T(n)=9T(n/3)+nT(n)=9T(n/3)+n

  • f(n)=nf(n)=n
  • a=9;b=3a=9;b=3, →→ thus nlogba=nlog39=Θ(n2)nlogb⁡a=nlog3⁡9=Θ(n2)

We can show that f(n)=O(nlog39−ϵ)f(n)=O(nlog3⁡9−ϵ), where ϵ=1ϵ=1.

  • the function nlogbanlogb⁡a is asymptotically larger than f(n)f(n)

Therefore case 1 of the master theorem applies and we have T(n)=Θ(n2)T(n)=Θ(n2)

Example 2:
Given recurrence:

T(n)=T(2n/3)+1=1⋅T(23n)+1=1⋅T(n/32)+1T(n)=T(2n/3)+1=1⋅T(23n)+1=1⋅T(n/32)+1

  • f(n)=1f(n)=1
  • a=1;b=3/2a=1;b=3/2, →→ thus nlogba=nlog3/21=n0=Θ(1)nlogb⁡a=nlog3/2⁡1=n0=Θ(1)

We can show that f(n)=Θ(nlog3/21)f(n)=Θ(nlog3/2⁡1)

  • the functions f(n)f(n) and nlog3/21nlog3/2⁡1 are asymptotically equal

Therefore case 2 of the master theorem applies and we have:

T(n)=Θ(nlog3/21⋅log2n)=Θ(n0⋅log2n)=Θ(1⋅log2n)=Θ(log2n)T(n)=Θ(nlog3/2⁡1⋅log2⁡n)=Θ(n0⋅log2⁡n)=Θ(1⋅log2⁡n)=Θ(log2⁡n)

Given recurrence:

T(n)=3T(n/4)+n⋅log2nT(n)=3T(n/4)+n⋅log2⁡n

  • a=3;b=4a=3;b=4, →→ thus nlogba=nlog43=n0.792=Θ(1)nlogb⁡a=nlog4⁡3=n0.792=Θ(1)

We can show that f(n)=Ω(nlog43+ϵ)f(n)=Ω(nlog4⁡3+ϵ), where ϵ≈0.2ϵ≈0.2.


Furthermore, we can show that the regularity condition holds:
For sufficiently large nn, we can show that:

a⋅f(n/b)=3⋅(n/4)log2(n/4)≤(3/4)nlog2n=c⋅f(n)a⋅f(n/b)=3⋅(n/4)log2⁡(n/4)≤(3/4)nlog2⁡n=c⋅f(n)

T(n)=Θ(f(n))=Θ(n⋅log2n)T(n)=Θ(f(n))=Θ(n⋅log2n)

Example 4:
Given recurrence:

T(n)=2T(n/2)+n⋅log2nT(n)=2T(n/2)+n⋅log2⁡n

  • f(n)=n⋅log2nf(n)=n⋅log2⁡n
  • a=2;b=2a=2;b=2, →→ thus nlogba=nlog22=n1=Θ(n)nlogb⁡a=nlog2⁡2=n1=Θ(n)

We can see that f(n)=Ω(nlog22)f(n)=Ω(nlog2⁡2) (f(n)f(n) is asymptotically larger than nlog22nlog2⁡2) and are therefore tempted to assume that case 3 applies.

Caution
Even tough f(n)=n⋅log2nf(n)=n⋅log2⁡n is asymptotically larger than nlog22=nnlog2⁡2=n, it is not polynomially larger


The ratio f(n)/nlogba=(n⋅log2n)/n=log2nf(n)/nlogb⁡a=(n⋅log2⁡n)/n=log2⁡n is asymptotically smaller than nϵnϵ, for any ϵ>0ϵ>0

  • →log2n=O(nϵ)→log2⁡n=O(nϵ)

Therefore, we cannot use the master theorem to solve this recurrence.
