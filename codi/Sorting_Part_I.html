Sorting (Part I)

The Problem:

We are given an array A[a1,a2,a3,...,an]A[a1,a2,a3,...,an] and we want to sort the elements in the array such that we get the permutation A′[a′1,a′2,a′3,...,a′n]A′[a1′,a2′,a3′,...,an′], where a′1≤a′2≤a′3≤...≤a′na1′≤a2′≤a3′≤...≤an′.

1. Search for the minimum & exchange elements

One possibility for solving the problem is searching for the minimum element aiai in our array AA and to exchange it with the first element in AA. Subsequently, we would repeat this procedure for A∖{ai}A∖{ai} (i.e. for AA excluding the first element) and exchange the new minimum with the second element in AA and so on.

The worst case would of course be when the numbers in the input array AA are sorted in reverse order. In this case wou would always have to search the full length of the remaining array to find the minimum.

Therfore, the worst-case running time of this algorithm is:

T(n)=O(n)+T(n−1)=n+(n−1)+T(n−2)=n+(n−1)+(n−2)+T(n−3)⋮=n+(n−1)+(n−2)+...+1+0=∑i=1niT(n)=O(n)+T(n−1)=n+(n−1)+T(n−2)=n+(n−1)+(n−2)+T(n−3)⋮=n+(n−1)+(n−2)+...+1+0=∑i=1ni

By now we kno that this is an aritmetic series which has the closed form:

∑i=1ni=n(n+1)2∑i=1ni=n(n+1)2

So we get the following:

T(n)=∑i=1ni=n(n+1)2=12⋅n⋅(n+1)=O(n)⋅O(n)=O(n2)T(n)=∑i=1ni=n(n+1)2=12⋅n⋅(n+1)=O(n)⋅O(n)=O(n2)

We have already come accross the Insertion Sort algorithm in a previous post.

Again, the worst case would be when the numbers in the input array AA are sorted in reverse order. In this case the inner while-loop (lines 6-9) would have to iterate the maximum number of times for each iteration of the outer for-loop.

3. Bubble Sort

The Bubble Sort algorithm also works very simple. It repeatedly iterates through the elements of the input array AA and swaps adjacent elements to put them into sorted order.












public void BubbleSort(int[] A) {
    for(int i = 0; i < A.length - 2; i++ ) {
        for(int j = A.length - 1; j > i ; j--) {
            if(A[j] < A[j-1]){
                int temp = A[j];
                A[j] = A[j-1];
                A[j-1] = temp;
            }
        }
    }
}

Analysis:

Similarly to the other sorting algorithms above, the worst case would be when the numbers in the input array AA are sorted in reverse order. In this case the inner for-loop (line 3) would have to iterate the maximum number of times for each iteration of the outer for-loop (line 2).This is the case because the condition of the if-statement (line 4) would always return true.

The worst-case running time of Bubble Sort is therefore also T(n)=O(n)+T(n−1)⟺T(n)=O(n)+T(n−1)⟺ T(n)=O(n2)T(n)=O(n2) (as shown above).
