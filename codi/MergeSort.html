The divide-and-conquer approach

General approach:

  • Divide: break down the problem into several smaller subproblems, that are similar to the original problem
  • Conquer: solve the subproblems recursively
  • Combine: combine their solutions to solve the original problem
  • Advantage: running time is often easily determined

Structure of Merge Sort

Divide:

  • Divide the nn-element sequence to be sorted into two subsequences of n/2n/2 elements each

Conquer:

  • Sort the two subsequences recursively using Merge Sort

The recursion terminates, when each sequence to be sorted has length 1 and therefore is already in sorted order

Combine:

  • Merge the two sorted subsequences to solve the overall sorting problem

The ‘Merge’ subprocedure

  • A is an array and left, middle and right are indices of the array A, such that left ≤≤ middle << right
  • the procedure assumes that the subarrays A[left…middle] and A[middle+1…right] are already in sorted order
  • since we perform at most nn basic steps, the ‘Merge’ subprocedure takes time Θ(n)Θ(n)


















































// auxiliary procedure for merging two sorted subarrays
public void Merge(int[] A, int left, int middle, int right) {
    //determine the size of the two temporary subarrays
    int n1 = middle - left + 1;
    int n2 = right - middle;
    //initialize two new temporary subarrays
    int[] Left = new int[n1];
    int[] Right = new int[n2];
    //copy data to the newly created temporary subarrays
    for(int i = 0; i < n1; i++) {
        Left[i] = A[left + i];
    }
    for(int j = 0; j < n2; j++) {
        Right[j] = A[middle + (j + 1)];
    }
    //initialize indices of temporary subarrays
    int i = 0;
    int j = 0;
    /*
    initialize the index of the array that is created by merging
    the two temporary subarrays
    */
    int k = left;
    //merge the two temporary subarrays
    while( i < n1 && j < n2) {
        if (Left[i] <= Right[j]) {
            A[k] = Left[i];
            i++;
        }
        else {
            A[k] = Right[j];
            j++;
        }
        k++;
    }
    /*
    in the case that one of the temporary subarrays is empty,
    simply add the remaining elements of the other subarray
     */
    while(i < n1){
        A[k] = Left[i];
        i++;
        k++;
    }
    while (j < n2) {
        A[k] = Right[j];
        j++;
        k++;
    }
}

The main ‘Merge Sort’ procedure

  • this procedure sorts the elements in the array subarray A[left…right]
  • if left ≥≥ right, the subarray has at most one element and is therefore already sorted
  • to sort the entire array AA, we make the initial call MergeSort(A,0,A.length-1)













//main procedure to sort the whole original array
public void MergeSort(int[] A, int left, int right) {
    if(left < right){
        // find the middle element of the array
        int middle = (left + right) / 2;
        //recursively sort first half of array
        MergeSort(A, left, middle);
        //recursively sort second half of array
        MergeSort(A, (middle + 1), right);
        //call to Merge() method to merge the two sorted subarrays
        Merge(A, left, middle, right);
    }
}

Analysis of Merge Sort

  • for simplicity we assume that the problem size is an exact power of 2, i.e. n=2kn=2k
  • Merge sort on a single element takes constant time, i.e. Θ(1)Θ(1)
  • Merge sort on n>1n>1 elements yields the following running time:

Divide:

  • this step simply computes the middle element of a given subarray and therefore only takes constant time, i.e. Θ(1)Θ(1)

Conquer:

  • this step recursively solves two subproblems of size n/2n/2 each and therefore contributes 2⋅T(n/2)2⋅T(n/2) to the running time

Combine:

  • as stated above, the Merge() procedure takes linear time Θ(n)Θ(n) on an nn-elemt array

→→ adding the contributions of the Divide and the Conquer steps, we get a total contribution of Θ(1)+Θ(n)=Θ(n)Θ(1)+Θ(n)=Θ(n) for both steps to the total running time.

We thus end up with:

T(n)={Θ(1), if n = 12⋅T(n/2)+Θ(n), if n > 1T(n)={Θ(1), if n = 12⋅T(n/2)+Θ(n), if n > 1

We can rewrite the recurrence like this:

T(n)={c, if n = 12⋅T(n/2)+cn, if n > 1T(n)={c, if n = 12⋅T(n/2)+cn, if n > 1

, assuming that the constant cc represents the time required to solve a problem of size n=1n=1 as well as the time per array alement of the Divide and Combine steps

  • the cncn term at the root represents the cost at the top level of the recursion
  • the costs of the two subtrees of the root are c(n/2)c(n/2) each

As we know that nn is an exact power of two (i.e. n=2kn=2k) and that in each recursive call to MergeSort(), the problem size is reduced to n/2n/2, we know that at some point we will reach the boundary condition of n=1n=1, for which T(n)=cT(n)=c (→→ represented by the bottom level of the recursion tree).

Adding the costs per level of the tree:

  • the top level has cost: cncn
  • the second level has cost: c(n/2)+c(n/2)=cnc(n/2)+c(n/2)=cn
  • the third level has cost: c(n/4)+c(n/4)+c(n/4)+c(n/4)=cnc(n/4)+c(n/4)+c(n/4)+c(n/4)=cn
  • …

In general we can say:

  • that level ii of the tree (below the top level, i.e. ii = 1 (second level), 2 (third level), 3, …) has 2i2i nodes
  • each node contributes a cost of: c(n/2i)c(n/2i)
  • therefore, each level ii below the top level has a total cost of:
    2i⋅c(n/2i)=cn2i⋅c(n/2i)=cn
  • the bottom level has nn nodes, each contributing a cost of cc for a total cost of cncn

How many levels does our recursion tree have?

  • the recursion terminates when we reach the boundary condition of n=1n=1
  • our original problem size is an exact power of two: n=2kn=2k
  • at each depth i (for i=0,1,2,3,...i=0,1,2,3,...) of the recursion tree, the problem size is reduced to n/2in/2i of the original size
  • →→ thus we reach the boundary condition (i.e. n=1n=1), when 2i=n2i=n which is the case at depth i for i=log2ni=log2⁡n

For a given input of size nn:

  • we now know that our recursion tree has a depth of log2nlog2⁡n
  • from this we can infer that our recursion tree has log2n+1log2⁡n+1 levels

Note the difference between the concepts of depth and levels. You can find a description of these concepts here. However, for now it is enough to know that the level of a node is its depth +1

The toatal cost of our recursion tree:

  • we simply add the cost accross all levels of the recursion tree
  • we have log2n+1log2⁡n+1 levels, each contributing a cost of cncn

Thus, we have a total cost of:

cn⋅(log2n+1)=cn⋅log2n+cncn⋅(log2⁡n+1)=cn⋅log2⁡n+cn

  • ignoring lower-order terms and the constant cc, we get:
    Θ(n⋅log2n)Θ(n⋅log2⁡n)

With a worst-case running time of T(n)=Θ(n⋅log2n)T(n)=Θ(n⋅log2⁡n) Merge Sort beats Insertion Sort, whose worst-case running time is T(n)=Θ(n2)T(n)=Θ(n2).
