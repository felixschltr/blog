4. Heapsort

The Heapsort algorithm makes use of a data structure called a 'Heap’, which is closely related to a binary tree. So before we’ll have a look at the algorithm itself, we will cover the basics on ‘Heaps’.

Heap Basics

  • a heap is an array object that we can represent as a complete binary tree
  • a binary tree is either empty, or is composed of a root node, a left subtree and a right subtree
  • in a complete binary tree all levels are completely filled except possibly the lowest one, which is filled starting from the left side.

⇒⇒ For a quick introduction to tree data structures have a look at this previous post. ⇐⇐

  • each node of the tree corresponds to/contains an element in the array

Height of a node

The height of a tree = the height of its root

Depth of a node

= the number of edges from the node to the root

Level of a node

= Depth of node + 1

Types of heaps:

There are two kinds of binary heaps, which are defined in terms of a “heap property”:

“Max-heaps”:

  • the values inside the nodes of a “max heap” satisfy the following “max-heap property”:
    A[parent(i)]≥A[i]A[parent(i)]≥A[i]

(i.e. the value inside a node ii is always smaller or equal to the value inside its parent node.)

In a max heap the maximum value is stored in the root node.

“Min heaps”:

  • the values inside the nodes of a “min heap” satisfy the following “min-heap property”:
    A[parent(i)]≤A[i]A[parent(i)]≤A[i]

(i.e. the value inside a node ii is always greater or equal to the value inside its parent node.)

In a min heap the minimum value is stored in the root node.

As you may have noticed Fig.1 is an example of a min heap, as it satisfies the min heap property and so the root node stores the minimum value.

Maintaining the heap property

Here is the algorithm of the subprocedure minHeapify() which is used to maintain the min-heap property (in case of a min heap):
























public static void minHeapify(int arr[], int n, int i) {
    //initialize the root node as the smallest
    int smallest = i;
    //initialize left and right children
    int left = 2 * i + 1; //equiv. to '2 * i' (index starts at 0)
    int right = 2 * i + 2;  //equiv. to '2 * i + 1' (index starts at 0)
    //check if left child is smaller than root
    if(left < n && arr[left] < arr[smallest]) {
        smallest = left;
    }
    //check if right child is smaller than root
    if(right < n && arr[right] < arr[smallest]) {
        smallest = right;
    }
    //exchange values in case one is smaller
    if (smallest != i) {
        int temp = arr[i];
        arr[i] = arr[smallest];
        arr[smallest] = temp;
        // Recursively heapify the affected sub-tree
        minHeapify(arr, n, smallest);
    }
}

Building a min heap

To build a min heap in a ‘bottom-up’ procedure , we make use of the previously explained property that the elements A[⌊n/2⌋,⌊n/2⌋+1,⌊n/2⌋+2,...n]A[⌊n/2⌋,⌊n/2⌋+1,⌊n/2⌋+2,...n] in our array representation of the heap are all leafs.

Problem:
We start out with an array A[0...n−1]A[0...n−1], where n=A.lengthn=A.length, which we want to transform into a min heap.

As the elements in A[⌊n/2⌋,⌊n/2⌋+1,⌊n/2⌋+2,...n]A[⌊n/2⌋,⌊n/2⌋+1,⌊n/2⌋+2,...n] are all leafs, we can look at them as min heaps of size 11 ,each (i.e they already fulfill the min-heap property).
Instead, we will concentrate on the remaining elements A[0...⌊n/2⌋−1]A[0...⌊n/2⌋−1].
The following buildMinHeap() procedure makes use of the minHeapify() subprocedure to build a min heap:






 public static void buildMinHeap(int[] arr) {
    for (int i = (arr.length / 2) - 1; i >= 0; i--) {
        minHeapify(arr, arr.length, i);
    }
}

Building a min heap takes time O(n)O(n). This may be unituitive, as it performs an operation that runs in time O(log2n)O(log2⁡n) (i.e. minHeapify()) n/2n/2 times, but you can have a look at the proof here or in the book Introduction to Algorithms, 3rd ed. Chapter 6.4, p. 159.

The Heapsort algorithm

The final Heapsort algorithm looks like this:













public static void heapSort(int[] arr) {
    //build min heap from input array
    buildMinHeap(arr);
    for(int i = arr.length -1; i > 0; i--) {
        // Move the current root to the end
        int temp = arr[0];
        arr[0] = arr[i];
        arr[i] = temp;
        // recursively call minHeapify() on the reduced heap
        minHeapify(arr, i, 0);
    }
}

The algorithm works like this:

  • first, a min heap is built from the input array by calling the buildMinHeap() procedure
  • next, the for-loop iterates backwards through the heap and moves the value stored at the root node (i.e. the minimum value) to the end of the heap
  • finally, the minHeapify() procedure is called recursively on the heap of size−1size−1 (i.e. excluding the value which has just been placed at the end of the heap)

Analysis

  • the buildMinHeap() procedure takes time O(n)O(n) (as stated above)
  • the minHeapify() procedure takes time O(log2n)O(log2⁡n) and is called n−1n−1 times

Taken together, we get the worst-case running time:

T(n)=O(n)+O(n⋅log2n)=O(n⋅log2n)T(n)=O(n)+O(n⋅log2⁡n)=O(n⋅log2⁡n)

Important notes

You may have noticed that the Heapsort algorithm presented above, unlike other sorting algorithms we have seen so far, sorts the input array in decreasing order insetad of increasing order.
