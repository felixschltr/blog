3 Methods for solving recurrences

  • solving recurrences = obtaining asymptotic ΘΘ or OO bounds on the solution

1. The Substitution method

  • we guess a bound and then use mathematical induction to prove our guess correct

2. The recursion-tree method

  • converts the recursion into a tree whose nodes represent the costs incurred at various levels of the recursion
  • we use techniques for bounding summations to solve the recurrence

3. The master theorem

  • provides bounds for recurrences of the form:
    T(n)=a⋅T(n/b)+f(n)T(n)=a⋅T(n/b)+f(n)

    , where a≥1a≥1, b>1b>1, and f(n)f(n) is a given function, arise frequently.

Equations of this kind characterize divide-and-conquer algotithms that create a subproblems, each of which is 1/b the size of the original problem, and in which the divide and combine steps together take f(n) time

Technicalities in recurrences

For recurrences like

  • →→ we assume that T(n)T(n) is constant for small n

We simply write:

T(n)=2⋅T(n/2)+Θ(n)T(n)=2⋅T(n/2)+Θ(n)

The substitution method

1. Guess the form of the solution

2. Use mathematicalinduction to find the constants and show that the solution works

The recursion-tree method

  • each node reprsents the cost of a single suboroblem somewhere in the set of recursive function invocations
  • we sum the costs within each level of the tree to obtain a set of per-level costs
  • we sum all the per-level costs to determine the total costs of all levels of the recursion

A recursion tree is best used to generate a good guess, which you can then verify by the substitution method

Intermission: Tree basics

Example Tree

Height of a node

= the number of edges on the longest path between the node and a leaf.

The height of a tree = the height of its root

Depth of a node

= the number of edges from the node to the root

  • the depth of the root is always 0

Levels of a tree

= 1 + the number of edges between a node and the root (i.e. Depth + 1)
