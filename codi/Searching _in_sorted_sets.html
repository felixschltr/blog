Searching in sorted sets

Linear Search

We are given a sorted array AA containing nn numbers:

  • i.e. A[0...n−1]A[0...n−1], where A[0]≤A[1]≤A[2]≤...≤A[n−1]A[0]≤A[1]≤A[2]≤...≤A[n−1]

We would like to know wheteher or not our array AA contains a certain number (i.e. numberToSearch).

We initialize a search index next = 0














// Linear Search algorithm
int[] A = new int[n];
int numberToSearch;
int next = 0;
while(numberToSearch > A[next]) {
        next += 1;
        }
if(numberToSearch == A[next]){
        System.out.println("\nNumber found in array at index: " + next);
    }
else {
        System.out.println("\nNumber not in array!");
    }

The algorithm works very simple. It scans through the array AA starting at index 0 and compares every number against the number we want to find (i.e. numberToSearch) by making use of our search index next (while-loop in lines 5-7).
When it finds that the next number in our array is no longer smaller than our numberToSearch, it compares the two numbers to determine whether the numberToSearch is contained in the array AA (if structure lines 8-13).

Analysis

The running time of the Linear Search algorithm is also simple to analyze:

The best-case running time is O(1)O(1) (constant time)

  • this is the case when our numberToSearch is the first element in our array

The worst-case running time is O(n)O(n) (linear time)

  • this is the case when our numberToSearch is not contained in our array, so the whole array has to be searched

The expected running time is also O(n)O(n) (linear time)

  • on average we expect that we will have to search n/2n/2 elements of our array until we find our numberToSearch (provided that it is part of the array). n/2=12⋅n=O(n)n/2=12⋅n=O(n)

Binary Search

The starting point is the same as for Linear Search:

We are given a sorted array AA containing nn numbers:

  • i.e. A[0...n−1]A[0...n−1], where A[0]≤A[1]≤A[2]≤...≤A[n−1]A[0]≤A[1]≤A[2]≤...≤A[n−1]

We would like to know wheteher or not our array AA contains a certain number (i.e. numberToSearch).

Additionally, we initialize three index variables: next, left and right.





























//the BinarySearch() method
public void BinarySearch(int[] A, int left, int right, int numberToSearch) {
        if((right - left) >= 0) {
            //determine middle element of array (called next)
            int next = left + (right - left) / 2;
            //compare middle element to numberToSearch
            if (A[next] == numberToSearch) {
                System.out.println("\nNumber found at index: " + next);
            }
            else if (A[next] > numberToSearch) {
                /*
                recursively call Binary Search() on lower
                subarray of length n/2
                */
                BinarySearch(A, left, next-1 , numberToSearch);
            }
            else {
                /*
                recursively call Binary Search() on upper
                subarray of length n/2
                */
                BinarySearch(A, next + 1, right, numberToSearch );
            }
        }
        else {
            System.out.println("\nNumber is not in the array!");
        }
    }

Interpolation Search

Interpolation Search works similar to Binary Search, the only difference being the way that the search index next is determined.

In Interpolation Search next is determined using the following formula:

next=left+⌊numberToSearch−A[left]A[right]−A[left]⋅(right−left)⌋next=left+⌊numberToSearch−A[left]A[right]−A[left]⋅(right−left)⌋

What does this formula mean?

Essentially, we are using the knowledge that the array we are searching (i.e. AA) comes in sorted order to determine where our numberToSearch might be relative to the first and the last number in our array.

An example of a real-world application of this method would be searching a telephone directory. When searching for a certain name in a telephone directory you would usually make use of the fact that the directory is lexicographically sorted. This means, that you would not open the directory at some random page. Instead, most people would open the directory at a page close to where they would expect the name, given the lexicographical order.






















 public void InterpolSearch(int[] A, int l, int r, int nts) {
    if((r - l) >= 0) {
        /*
        note the different formula to determine 'next'
        compared to BinarySearch()
        */
        int next = l + ((nts - A[l])/(A[r] - A[l])) * (r - l);    
        if (A[next] == nts) {
            System.out.println("\nNumber found at index: " + next);
        }
        else if (A[next] > nts) {
            InterpolSearch(A, l, next-1 , nts);
        }
        else {
            InterpolSearch(A, next + 1, r, nts );
        }
    }
    else {
        System.out.println("\nNumber is not in the array!");
    }
}

Interpolation Search has a good average running time of O(log2log2n)O(log2⁡log2⁡n).
The worst-case running time is O(n)O(n)
